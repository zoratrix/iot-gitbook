# Таблица параметров

Каждый реальный и виртуальный объект в системе имеет определенные **параметры (свойства)** - **входные** и **выходные**, значениями которых он обменивается с другими объектами. Одной из частей документации к проекту является **таблица параметров** задействованных в системе объектов.&#x20;

Таблица в общем виде примерно такая:

| параметр (название) | описание                                     | input/output | тип данных | диапазон значений |
| ------------------- | -------------------------------------------- | ------------ | ---------- | ----------------- |
| s                   | статус робота (0 = стоит, 1 = двигается)     | input        | int        | 0...1             |
| temp                | температура воздуха в помещении              | input        | float      | 0...35            |
| info                | информация о состоянии светофора             | input        | string     | не применимо      |
| btn1                | состояние кнопки (1 = нажата, 0 = не нажата) | input        | int        | 0...1             |
| x                   | координата Х                                 | output       | int        | -100...+320       |
| color               | цвет светодиодов ленты                       | output       | byte       | 0...255           |

**Под входными/выходными значениями понимается следующее:** мы создаем систему управления верхнего уровня. **Входные значения (input)** - это значения мониторинга, то есть те, которые отслеживаются системой, приходят от оборудования на сервер (датчики, кнопки, положение моторов текущее и т.д.). Параметры **выходные (output)** - это те значения, которые отправляются системой на оборудование для управления (положение робота по оси Y, цвет светодиода, включить лампу 3 и т.д.)&#x20;

Диапазон значений может быть выбран по-разному.&#x20;

* Для каких-то значений, например color  в таблице выше, теоретически существует бесконечное возможное число значений, но диапазон ограничен выбранным типом данных (byte от 0 до 255).
* Для других параметров, например, s - статус робота, возможные значения определены программистами, которые написали прошивку робота. Вы узнаете этот диапазон из задания или инструкции к оборудованию
* Аналогично, значения диапазона для temp - температуры - зависят от того датчика, который эту температуру измеряет. И даже если физически на улице температура может опуститься и ниже 0 и подняться выше 35, если в инструкции указан диапазон 0...35, значит за его пределами датчик не работает и иных значений вы получить не можете
