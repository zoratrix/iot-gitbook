# 7 События-триггеры

События триггеры - это такие события, при которых на дашбордах происходят какие-то действия и изменения. Вам нужно перечислить их для каждого дашборда:

1. **События триггеры для дашборда инженера-технолога**
   1. Пользователь открыл дашборд -> отобразить все виджеты
   2. Пришли данные от оборудования -> вывести данные мониторинга
   3. Значение мониторинга робота вышло за границы допустимых -> раскрасить значение в желтый
   4. Значение мониторинга робота вышло за границы критических -> раскрасить значение в красный
2. **События триггеры для дашборда оператора**
   1. Пользователь открыл дашборд -> отобразить все виджеты
   2. Пользователь открыл дашборд -> передать начальные координаты в робота1 и робота2
   3. Роботы переместились в новые координаты -> отобразить новые значения X, V&#x20;
   4. Пользователь выбрал код сборки -> вывести статус кода (верный/неверный) и план сборки
3. **События триггеры для дашборда менеджера производства**
   1. Пользователь открыл дашборд -> отобразить все виджеты
   2. Пользователь включил вывод графиков -> вывести графики
   3. Пользователь выбрал код сборки -> вывести статус кода (верный/неверный) и план сборки

**Вы можете дополнить список своими событиями. Должно быть хотя бы по 2-3 триггера для каждого дашборда!**
