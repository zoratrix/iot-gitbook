# Метеостанция по сети

Задание: написать программу, которая будет отправлять температуру числом в топик **"smena/X/temperature"**, а относительную влажность в топик **"smena/X/humidity"** раз в 3 секунды _(для таймера не использовать delay()!)._

Отправка сообщений в топик выполняется командой:

```arduino
client.publish(топик_куда_отправляем, что_отправляем);
```

Отправку выполнять в цикле loop();

Важно: отправлять с помощью функции publish можно сообщения типа c\_str. Влажность и температура у нас изначально - это числа, причем нецелые (тип float). Поэтому нужно выполнить преобразование float -> c\_str (подробно про строки разницу между их типами еще раз: [String-строки](https://alexgyver.ru/lessons/strings/), [Си-строки](https://alexgyver.ru/lessons/cstring/)). Напрямую такое преобразование сделать не получится: сначала нужно преобразовать число float в String, а затем из String в c\_str.

Для этого создайте в начале кода вместе с переменными float temp; float hum; также переменные для значений температуры и влажности, но строками ("текстом"): String temp\_string и String hum\_string.

Далее, там же в коде, где вы выполняете получение данных с датчика:

```arduino
temp = dht_sensor.readTemperature();
hum = dht_sensor.readHumidity();
```

добавьте преобразование в String и запись в новые переменные:

```arduino
temp = dht_sensor.readTemperature();
hum = dht_sensor.readHumidity();

temp_string = (String)temp;
hum_string = (String)hum;
```

(конечно, при желании можно это сделать без лишних переменных одной строкой, если хотите - попробуйте самостоятельно)

Там, где вы будете отправлять данные командой client.publish() выполните преобразование String->c\_str:

```arduino
client.publish(название_топика, temp_string.c_str());
```

(для влажности аналогично)

<summary>Подсказки</summary>

* **не забудьте создать топики в самом начале**&#x20;
* функция setup\_wifi() отвечает за подключение к WiFi -она не интересует нас;&#x20;
* функция callback() отвечает за реакцию на пришедшие сообщения, а мы должны отправлять сообщения, то есть тоже не подходит.&#x20;
* Функция reconnect() отвечает за подключение к MQTT, тоже не то. Но в ней можно найти пример, как опубликовать сообщение
* функция setup() выполняется 1 раз при запуске, не то
* функция loop() выполняется постоянно по кругу. Можно попробовать добавить публикацию сообщений в ней
* **Но не забудьте про таймер!**
