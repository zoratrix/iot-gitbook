# Blink



Для того, чтобы проверить плату, не нужно дополнительного оборудования. На Wemos Mini уже есть светодиод, который можно заставить мигать.

Скорее всего, вы уже знакомы с тем, как выглядит программа для мигания светодиода для микроконтроллера, но в этом разделе повторим все еще раз.&#x20;

В простейшем случае код программы для контроллеров Arduino и ESP32 (наш случай) делится на три основные части.

<figure><img src="../.gitbook/assets/image (87).png" alt=""><figcaption></figcaption></figure>

Часть A, которая находится в самом начале кода:

* Выполняется 1 раз при запуске
* Назначить имена и адреса пинов
* Подключить библиотеки
* Инициализировать переменные

Часть B, функция `void setup()`:

* Выполняется 1 раз при запуске
* Назначить режимы работы пинов
* Прописать настройки (например логин и пароль от WiFi)

Часть C, функция `void loop()` - основной  код программы:

* Выполняется постоянно
* Команды, которые будут выполняться, пока контроллер включен

Основные правила синтаксиса, справедливые для всего кода:

* Каждая строчка заканчивается точкой с запятой `;`
* Если команда написана правильно, то она выделяется цветом
* Есть разница между строчными и заглавными буквами!

Переходим к программе для мигания. Если бы мы подключили какой-то элемент, например отдельный светодиод, то в начале кода удобно было бы прописать, куда подключен этот светодиод и придумать ему название с помощью команды `#define`. Далее в коде к подключенному устройству можно обращаться по назначенному имени или по номеру пина, в который это устройство подключено. Наш встроенный светодиод отдельно обозначать не нужно - он уже известен в коде как `LED_BUILTIN`.

При работе напрямую с пинами платы в цикле функции `setup()` нужно объявить, в каком режиме используется этот пин - как входной или выходной.&#x20;

{% hint style="info" %}
**Вход** контроллера позволяет ему **получать** информацию от какого-то элемента. Ко входам подключаются все датчики всех типов и выключатели. Выключатель или датчик даёт сигнал о нажатии или о срабатывании, контроллер его получает, значит, для контроллера это вход.

Контроллер не посылает на датчики и выключатели никаких сигналов, только получает сигналы от них.
{% endhint %}

{% hint style="info" %}
**Выход** контроллера позволяет контроллеру выдавать сигнал. То есть, управлять чем-то. Например, светом, шторами, приводами радиаторов, сиреной, дверным замком, розетками - чем угодно, подавая на него напряжение или замыкая контакты
{% endhint %}

Для этого используется функция `pinMode(название или номер пина, режим работы)`. Режим работы - INPUT (входной) или OUTPUT (выходной). Получается команда `pinMode(LED_BUILTIN, OUTPUT);`

Далее в цикле `loop()` нужно написать, что, собственно, должен делать этот светодиод. Мигание - это попеременное включение\выключение с небольшой паузой между ними. Чтобы включить светодиод, нужно отправить на пин, к которому он подключен, "высокий" электрический сигнал (логическую 1 или true). Чтобы выключить, нужно отправить "низкий" сигнал (логический 0 или false).&#x20;

Чтобы отправить на пин какой-нибудь сигнал, используется функция `digitalWrite(название пина, какой сигнал отправить)`. Чтобы выключить встроенный светодиод, нужно написать:

`digitalWrite(LED_BUILTIN, 0);`

или

`digitalWrite(LED_BUILTIN, LOW);`

Соответственно, чтобы включить:

`digitalWrite(LED_BUILTIN, 1);`

или

`digitalWrite(LED_BUILTIN, HIGH);`

После включения и после выключения нужно поставить паузы, например в 1 секунду. Пауза в простейшем (но не самом удобном) случае может быть сделана с помощью команды delay(время в милисекундах)

В результате, весь код будет выглядеть следующим образом:

```clike
void setup() {
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop() {
  digitalWrite(LED_BUILTIN, 1);  // включаем светодиод
  delay(1000);                   // ждем 1 секунду
  digitalWrite(LED_BUILTIN, 0);  // выключаем светодиод
  delay(1000);                   // ждем 1 секунду
}
```

Если все в порядке, то&#x20;

Когда всё установлено и настроено, надо:

1.  Выбрать последовательный порт, к которому подключен контроллер.

    ![arduino\_select\_port](https://aiu.susu.ru/assets/images/iot/arduino/u/arduino_tools_port.png)

    Если портов несколько и вы не уверены в том, который нужен то спросите у инструктора, но скорее всего это будет **/dev/ttyUSB0**

    ![arduino\_port\_dev\_tty\_usb](https://aiu.susu.ru/assets/images/iot/arduino/u/arduino_tools_port_tty_usb0.png)
2.  Нажать кнопку **Загрузка** для компиляции и записи программы в контроллер.

    ![blink\_compile](https://aiu.susu.ru/assets/images/iot/arduino/u/blink_compile.png)
3.  Если всё прошло хорошо и нет сообщений об ошибке, то скетч записался в память контроллера

    ![blink\_downloading\_compleate](https://aiu.susu.ru/assets/images/iot/arduino/u/blink_downloading_compleate.png)

    а синий светодиод на нём стал периодически мигать.

    ![blink\_animated](https://aiu.susu.ru/assets/images/iot/wemos_blink.gif)
