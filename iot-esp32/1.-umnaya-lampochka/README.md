#### Библиотеки и переменные

Во-первых, нужно подключить несколько библиотек:

```arduino
#include <ESP8266WiFi.h>
#include <PubSubClient.h>
```

Библиотека ESP8266WiFi нужна для подключения к WiFi, PubSubClient - для работы по MQTT.&#x20;

Для работы нам понадобятся некоторые параметры, которым удобнее всего назначить имя и значение в начале кода (чтобы их было легко поменять, в случае чего, и не пришлось искать их по всей программе). Вот эти параметры:

* логин и пароль от сети WiFi
* адрес MQTT брокера (ip адрес вашего компьютера)
* порт MQTT брокера
* логин и пароль MQTT брокера
* топики, с которыми мы будем работать

Чтобы записать их в коде, используем команду `#define`

#### Функции

{% hint style="info" %}
Здесь небольшое отступление о том, что такое функции и как их использовать. Использованы материалы AlexGyver.
{% endhint %}

**Функция** – часть программы, блок кода, имеющий своё название. Большая программа может строиться из нескольких функций, каждая из которых выполняет свою задачу, поэтому можно назвать функцию **подпрограммой**. Использование функций очень сильно упрощает написание и чтение кода, и в большинстве случаев делает его оптимальным по объёму занимаемой памяти. Функция должна быть **описана**, и после этого может **вызываться**. Функция должна быть описана **снаружи других функций!** В общем виде функция имеет следующую структуру:

```
тип_данных имя_функции (аргументы) {
  тело_функции
}
```

Где **тип данных** – это тип данных, который **возвращает** функция, **имя функции** – имя, по которому функция **вызывается**, набор аргументов (параметров) – необязательный набор переменных, и **тело функции** – код, который будет выполняться.

Вызов функции осуществляется именем функции и передачей набора аргументов, если таковые имеются:

```
имя_функции(аргумент1, аргумент 2, аргументN);
```

Если аргументы не передаются, в любом случае нужно указать пустые скобки!

```
имя_функции();
```

Функция в C++ всегда **возвращает** результат. Этот термин означает, что после выполнения функции она выдаёт некое значение, которое можно присвоить другой переменной.

```
результат = имя_функции();
```

Функция может принимать аргументы, может не принимать, может возвращать какое-то значение, может не возвращать. Давайте рассмотрим эти варианты.

<details>

<summary>Функция, которая ничего не принимает и ничего не возвращает (void)</summary>

Самый простой для понимания вариант, с него и начнём. Помимо стандартных типов данных, есть ещё один – `void`, который переводится с английского как “пустота”. Создавая функцию типа void мы указываем компилятору, что никаких значений возвращаться не будет (точнее будет – функция вернёт “ничего”).&#x20;

В некоторых языках программирования есть понятие “процедура” – это как раз и есть функция, которая ничего не принимает и не возвращает. Но в C++ такого понятия нет, есть только функции.

Давайте напишем функцию, которая найдёт сумму двух чисел и присвоит её третьему числу. Так как функция у нас без аргументов и ничего не возвращает, переменные придётся объявить заранее и сделать их глобальными, иначе функция не будет иметь к ним доступ и мы получим ошибку:

```arduino
byte a, b;
int c;

void setup() {
  a = 10;
  b = 20;
  sumFunction();
  // после вызова функции
  // с имеет значение 30
}

void loop() {
}

void sumFunction() {
  c = a + b;
}
```

Это очень плохой пример с точки зрения оптимальности кода, но далее мы будем этот пример улучшать и в итоге получим конфетку. Чем он плох на данном этапе: у нас используются глобальные переменные, которые работают внутри функции, а одним из главных принципов программирования является разделение данных и действий. Будучи новичком, не стоит об этом сильно задумываться, позже вы сами к этому придёте. Разделяя **данные** и **действия** можно создавать универсальные инструменты, рассмотренная выше функция не является универсальной: она складывает глобальную `а` с глобальной `b` и записывает результат в глобальную же `c`. Сделаем следующий шаг к оптимизации: пусть функция **возвращает** значение.

</details>

<details>

<summary>Функция, которая ничего не принимает и возвращает результат</summary>

Чтобы функция могла вернуть численное значение, она должна быть описана с типом данных, который будет возвращаться. Нужно заранее подумать, какой тип будет возвращён, чтобы избежать ошибок. Например я знаю, что моя суммирующая функция работает с типом данных `byte`, она складывает два таких числа. Это означает, что результат вполне может превысить предел типа `byte` (сложили 100 и 200 и получили 300), значит функции следовало бы возвращать например тип данных `int`. Собственно поэтому у переменной `c` из предыдущего примера тип данных `int`.

Для возврата значения нам понадобится оператор `return`, который и будет возвращать число. Здесь нужно запомнить, что `return` не просто возвращает значение, а также **завершает выполнение функции**, то есть действия, указанные после `return`, выполнены уже не будут! На самом деле это очень удобно, ведь с помощью логических конструкций и операторов выбора можно предусмотреть в функции несколько разных `return`, которые будут возвращать разные значения или просто завершать выполнение функции.&#x20;

_Функция типа void вроде бы ничего не возвращает, но использование return; позволяет завершить выполнение функции по условию или как-то ещё. Это очень удобно!_

Давайте перепишем наш код так, чтобы числа `a` и `b` складывались и результат возвращался функцией, и этот результат мы уже “ручками” приравняем к c.

```arduino
byte a, b;
int c;
void setup() {
  a = 10;
  b = 20;
  c = sumFunction();
  // с имеет значение 30
}
void loop() {
}
int sumFunction() {
  return (a + b);
}
```

Ну вот, функция стала чуть более универсальной. Теперь результат сложения `a` и `b` как функцию можно использовать в других местах и приравнивать к другим переменным. Чтобы сделать код ещё более универсальным, давайте передавать величины для сложения в виде аргументов.

</details>

<details>

<summary>Функция, которая принимает аргументы и возвращает результат</summary>

Что касается аргументов, то они перечисляются в скобках через запятую с указанием типов данных. При вызове функции указанные аргументы превращаются в **локальные переменные**, с которыми можно работать внутри функции. Иными словами – это **копии** переданных переменных! При вызове функции эти переменные получают значения, которые мы указываем **при вызове**. Смотрим:

```arduino
byte a, b;
int c;
void setup() {
  a = 10;
  b = 20;
  c = sumFunction(a, b);
  // с имеет значение 30
}
void loop() {
}
int sumFunction(byte paramA, byte paramB) {
  return (paramA + paramB);
}
```

И вот так мы получили универсальную функцию `sumFunction`, которая принимает две величины типа `byte`, складывает их и возвращает результат типа `int`. Это и есть выполнение концепции “отделение кода от данных”, функция живёт сама по себе и не зависит от других переменных! Казалось бы, можно использовать функцию как `sumFunction(100, 200)`, и она вернёт значение `300`. Но не всё так просто, потому что **любое целое число в программе обрабатывается компилятором как int**, и при попытке передать такой вот `int` в нашу функцию, которая принимает `byte`, мы получим ошибку "нельзя передать `int` вместо `byte`". В этом случае можно привести тип числа к `byte` вручную, вот так это будет выглядеть:

```arduino
int c;

void setup() {
  c = sumFunction((byte)100, (byte)200);
  // с = 300
}

void loop() {
}

int sumFunction(byte paramA, byte paramB) {
  return (paramA + paramB);
}
```

Но лучше всё-таки сделать функцию более универсальной, пусть она принимает&#x20;

`int paramA` и `int paramB`, ведь данные переменные являются локальными, то есть создадутся при вызове функции и удалятся из памяти при завершении работы функции, и их “размер” в принципе не играет роли.

А как быть, если мы хотим складывать уже имеющийся функцией другие типы данных? Например, `float`. Можно преобразовать типы данных при передаче аргументов, но функция всё равно вернёт целое число. Сделать нашу функцию ещё более универсальной сможет такая штука C++ как **перегруженная функция**.

</details>

<details>

<summary>Перегруженные функции</summary>

Перегруженная функция это такая функция, которая определена несколько раз **с одинаковым именем**, но **разным возвращаемым типом** данных и/или **разным набором аргументов**.

```arduino
int c;
float d;

void setup() {
  float af = 5.5;
  float bf = 0.25;
  Serial.begin(9600);
  c = sumFunction(10, 20);      // результат 30
  c = sumFunction(10, 20, 30);  // результат 60
  d = sumFunction(af, bf);      // результат 5.75
  Serial.println(c);
  Serial.println(d);
}

void loop() {
}

int sumFunction(int paramA, int paramB) {
  return (paramA + paramB);
}

int sumFunction(int paramA, int paramB, int paramC) {
  return (paramA + paramB + paramC);
}

float sumFunction(float paramA, float paramB) {
  return (paramA + paramB);
}
```

Итак, у нас теперь целых три функции с одинаковым именем, но разными наборами аргументов и типами возвращаемого значения. Программа **сама разберётся** какую из функций использовать на основе передаваемых аргументов. Передали два `float` – работает третья функция, вернёт `float`. Передали три `int` – получили их сумму при помощи второй по счёту функции. Передали два `int` – получили их сумму при помощи первой функции. Вот такая удобная штука! Ещё одним вариантом перегруженной функции является **шаблонная функция**, она позволяет работать с данными любого типа в рамках одной функции, без перегрузов. Читайте ниже.

</details>

<details>

<summary>Типы данных</summary>

Функция может возвращать и принимать как аргумент _любые типы данных в любых сочетаниях_: целые числа, float, String, указатели, struct, enum и так далее. Например следующая функция принимает два числа, склеивает их в строку через запятую и возвращает:

```arduino
String toStr(int val1, float val2) {
  return (String)val1 + ", " + val2;  
}
```

При вызове `Serial.println(toStr(10, 3.14))` увидим в мониторе порта `10, 3.14`

</details>

#### Подключение к WiFi

Во-вторых, нам нужно будет подключиться к WiFi. Собственно алгоритм подключения к WiFi выглядит следующим образом:

* Запускаем соединение, для этого вызываем `WiFi.begin(название сети, пароль)`
* Если подключение прошло успешно, результат функции `WiFi.status()` должен стать `WL_CONNECTED`

То есть, запускаем подключение с помощью WiFi.begin() по кругу (с паузой в 500 мс) и как только подключение произошло, выходим из цикла. Это реализуем с помощью цикла while - в круглых скобках указываем условие, при выполнении которого цикл будет повторяться. Как только условие перестанет выполняться, цикл прервется.

Подробнее можно прочитать в документации на библиотеку [здесь](https://arduino-esp8266.readthedocs.io/en/3.1.2/esp8266wifi/readme.html), или у AlexGyver'a [здесь](https://kit.alexgyver.ru/tutorials/wemos-basic/).

```arduino
WiFi.begin(wifi_ssid, wifi_pass);
while (WiFi.status() != WL_CONNECTED) {
    delay(500);
 }
```

Однако, хотелось бы, чтобы все эти действия сопровождались каким-то текстом, который будет выводиться в терминал. Для вывода строки в терминал используется команда `Serial.println("ваш текст")`. Но для того, чтобы в терминале вообще стало появляться что-то, что вы отправите, в цикле setup() нужно будет запустить передачу данных в терминал. Это сделаем позже. Пока добавим следующее:

* После включения платы появляется строка <mark style="background-color:blue;">"Connecting to <название сети> <пароль сети>"</mark>&#x20;
* Пока выполняется подключение, на экране появляется строка загрузки из точек: <mark style="background-color:blue;">.....</mark> (и постепенно прибавляются новые точки)
* После успешного подключения выводятся надписи <mark style="background-color:blue;">"WiFi connected"</mark> и строка <mark style="background-color:blue;">"IP address: \<ip адрес контроллера>"</mark>

Если добавить все перечисленное, получится следующий код:

```arduino
  Serial.println();                       // пустая строка
  Serial.print("Connecting to ");
  Serial.println(wifi_ssid);              // выводим название сети WiFi
  Serial.println(wifi_pass);              // выводим пароль
  
  WiFi.begin(wifi_ssid, wifi_pass);       // подключение к WiFi
  while (WiFi.status() != WL_CONNECTED) { // проверка подключения к wifi
    delay(500);                           // задержка перед новым подключением
    Serial.print(".");                    // пока выполняется подключение, в терминале будет появляться линия загрузки из точек
    WiFi.printDiag(Serial);               // вывод диагностической информации о WiFi подключении
  }

  randomSeed(micros());                   // micros() - текущее время с момента запуска программы, записываем его в randomSeed для системных нужд 

  Serial.println("");                     // пустая строка
  Serial.println("WiFi connected");  
  Serial.println("IP address: ");
  Serial.println(WiFi.localIP());         // выводим IP контроллера
```

Все это отлично работает, но для удобства лучше всего "упаковать" это в одну функцию и дальше в коде вызывать подключение к WiFi одной строкой:

#### Подключение к MQTT-брокеру

С подключением MQTT все точно так же. Нам нужно запускать соединение с использованием ранее указанных параметров до тех пор, пока контроллер н подключится к брокеру, и сопровождать все это комментариями в терминале для удобства пользователя.

Как и в случае c WiFi, удобнее всего будет написать это в отдельной функции, которую мы потом будем вызывать. Создадим ее, пока пустую:

```arduino
void reconnect() {

}
```

Непосредственно подключение к брокеру осуществляется следующей командой: `client.connect(id клиента в формате cstr, логин mqtt, пароль mqtt)`

Данные именно сети, по которой мы устанавливаем подключение (в нашем случае WiFi) не нужны - потому что когда мы объявляли клиента MQTT в самом начале кода, мы уже это учли. Логин (<mark style="background-color:blue;">mqtt\_login</mark>) и пароль (<mark style="background-color:blue;">mqtt\_pass</mark>) MQTT нам известны. Неизвестен id клиента - его можем задать сами. Пусть id будет выглядеть как <mark style="background-color:blue;">"ESP8266Client-<внутренний номер платы>"</mark>. Этот внутренний номер платы можно получить командой ESP.getChipId(). Чтобы получился полный Client ID, нужно написать в коде:

```arduino
String clientId = "ESP8266Client-";  // начало строки, у всех общее
clientId += String(ESP.getChipId()); // += значит, что к тому, что уже записано в переменной (clientId) мы добавляем то, что находится после +=, то есть номер контроллера 
```

Тогда строка подключения к MQTT брокеру будет выглядеть как:

```arduino
client.connect(clientId.c_str(), mqtt_login, mqtt_pass)
```

Но не все так просто. Если подключение произошло, то `client.connect(clientId.c_str(), mqtt_login, mqtt_pass)` будет равно <mark style="background-color:blue;">true</mark>. Тогда можно вывести в терминал надпись об успешном подключении. Если подключение не произошло (то есть команда подключения = false), нужно попробовать еще раз, например, через 5 секунд.

Алгоритм получается примерно таким:

```
if (подключение произошло) {
    вывести надпись, что все супер;
    начать принимать данные из топиков
} else {
    вывести надпись, что подключение не произошло и попробовать еще раз
}
```

В виде кода:

```arduino
if (client.connect(clientId.c_str(), mqtt_login, mqtt_pass)) {  
      Serial.println("Connected");                       //выводим надпись "подключено"
      client.publish(mqtt_topic_status, "reconnected");  // публикуем в топик "статус" сообщение о переподулючении
      client.subscribe(mqtt_topic_led);                   // подписываемся на топик входящих сообщений (у нас он будет использоваться для управления лампой)
    } else {
      Serial.print("failed, rc=");                       // сообщение о том, что подключение неуспешно
      Serial.print(client.state());                      // сообщение с информацией о клиенте

      delay(5000);                                       // задержка в 5 сек перед новым подключением
    }
```

Когда нужно подключаться? Когда мы не подключены. Поэтому весь алгоритм подключения нужно "завернуть" в цикл while с условием `!сlient.connected()`, то есть цикл будет повторяться, пока клиент НЕ подключен.

Последняя дополнительная функция, которая нам нужна - функция для работы с данными из топиков. Мы помним, что общение по MQTT строится следующим образом - создаются топики, в которые **Издатели** публикуют сообщения, а **подписчики** - читают их. В нашем случае издателем будет программа на компьютере (мы вручную будем отправлять сообщения в топик), а подписчиком - плата Wemos Mini, которая, в свою очередь, в зависимости от содержания сообщения в топике, будет включать или выключать светодиод. Для реализации этого нужен по крайней мере один топик. Мы придумали его заранее - "`smena/X/led1`" и назвали <mark style="background-color:blue;">mqtt\_topic\_led</mark> для удобства использования в коде.

Создадим функцию callback. Ее особенностью будет наличие **аргументов:**&#x20;

* topic типа char - это название топика, в который пришло сообщение
* payload типа byte - это содержание сообщения
* length типа unsigned int - это длина сообщения

Вообще, такое объявление функции callback при использовании библиотеки PubSubClient является стандартным и описан в[ документации библиотеки](https://pubsubclient.knolleary.net/api#callback).

```arduino
void callback(char* topic, byte* payload, unsigned int length) {
  // здесь будет код функции
}

```

Пусть сначала система оповестит подписчика о том, что пришло сообщение, и куда оно пришло (в каком топике опубликовано). В терминал выведем надпись "Message arrived" с помощью Serial.print() и название топика (оно хранится в переменной topic, которая является аргументом), а также само сообщение, причем посимвольно с помощью цикла for (это связано с особенностями библиотеки)

```arduino
void callback(char* topic, byte* payload, unsigned int length) {
  
  Serial.print("Message arrived [");
  Serial.print(topic); // выводим название топика
  Serial.print("] ");
  for (int i = 0; i < length; i++) {
    Serial.print((char)payload[i]); // выводим посимвольно содежание сообщения
  }
  Serial.println(); // переход на новую строку

}
```

Наконец, с полученным значением в топике управления светодиодом нужно что-то делать. А именно, если отправили 0 - выключить светодиод, если отправили 1 - включить.

Издатель может отправлять сообщения в любой топик, может "придумывать" каждый раз новый, например `smena/X/led1`, `smena/X/led2`, `smena/X/naruto`, но мы определили, что для управления светодиодом должен использоваться топик `smena/X/led1`, поэтому подписчику и читать нужно сообщения, которые приходят только в него. Поэтому нужно сравнить название топика, в который пришло сообщение, с названием топика, котоый мы назначили для светодиода.&#x20;

Сделать это можно с помощью функции `strcmp(стока1, стока2)`. Она сравнивает коды всех символов в строках-аргументах (строка1 и строка2) и **выдает 0, если строки одинаковые**. Если строка1 больше строки2, то результат будет >0, если строка 2 больше строки 1, то результат будет <0. Но нам достаточно знать, что при strcmp(стока1, стока2) == 0 строки одинаковые, то есть, если в качестве сток мы введем названия топиков, то сможем проверить, пришло ли сообщение в топик управления светодиодом `mqtt_topic_led` или в какой-то другой.

<strong>    // тут будет код
</strong>}
</code></pre>

Наконец, если пришла 1, то мы должны включить светодиод, если 0 - выключить. Важно отметить, что все сообщения - это строки, поэтому при сравнении 0 и 1 нужно указывать как символы, в кавычках "".

Переменная payload хранит пришедшее сообщение. У нас сообщение должно состоять из 1 символа (0 либо 1), поэтому нас интересует первый символ в строке payload. Так как строка - это массив символов, чтобы обратиться к первому символу в этом массиве, укажем payload\[0] (ноль, так как нумерация в массивах начинается с 0). Дополнительно пеобразуем это в тип char. В результате можем написать следующий код:

#### Функция Setup

Теперь переходим к функции `void setup(){}`

Так как мы будем управлять светодиодом (используем встроенный, его не надо объявлять в начале, он называется `BUILTIN_LED`), нужно указать режим работы пина, к которому этот светодиод подключен - `OUTPUT` (выходной) с помощью команды `pinMode()`.&#x20;

{% hint style="info" %}
Важно: на наших платах для встроенного светодиода 1 (HIGH) - это выключить, а 0 (LOW) - это "включить".&#x20;
{% endhint %}

```arduino
void setup() {
  pinMode(BUILTIN_LED, OUTPUT);  // инициализируем пин встроенного светодиода как выход
 
}
```

Далее запускаем передачу данных в терминал со скоростью 115200

```arduino
void setup() {
  pinMode(BUILTIN_LED, OUTPUT);     // инициализируем пин встроенного светодиода как выход
  
  Serial.begin(115200);             // запускаем передачу данных в терминал
  
}
```

После этого нужно подключиться к WiFi. Вызываем ранее написанную нами функцию `setup_wifi()`. Мы делаем это внутри цикла setup, так как подключиться к сети нужно один раз, это не нужно повторять постоянно во время работы программы. Если подключиться не получится, функция `setup_wifi()` сама будет делать это по кругу, пока не получится, и программа дальше не пойдет.

```arduino
void setup() {
  pinMode(BUILTIN_LED, OUTPUT);     // инициализируем пин встроенного светодиода как выход
  
  Serial.begin(115200);             // запускаем передачу данных в терминал
  setup_wifi();                     // подключаемся к wifi
}
```

После также нужно установить соединение с MQTT брокером. Делаем это командой `client.setServer(mqtt_server, mqtt_port)`.

Также нужно указать, какая функция будет обрабатывать сообщения командой `client.setCallback(функция обработки сообщений)`. Наша функция называется `callback`, поэтому в аргументы вписываем ее. Итоговый код функции `setup`:

```arduino
void setup() {
  pinMode(BUILTIN_LED, OUTPUT);     // Initialize the BUILTIN_LED pin as an output
  //WARNING: BUILTIN_LED has pullup resistor, HIGH = OFF, LOW = ON
  
  Serial.begin(115200);             // запускаем передачу данных в терминал
  setup_wifi();                     // подключаемся к wifi
  client.setServer(mqtt_server, mqtt_port); // подключаемся к брокеру
  client.setCallback(callback);
}
```

#### Функция loop

Внутри функции loop нам нужно запустить подключение клиента к брокеру с помощью функции `reconnect()`, причем выполнять это подключение нужно, если клиент НЕ подключен, то есть если `!client.connected() == true`.&#x20;

Если клиент подключен, идем дальше и запускаем общение с помощью команды `client.loop()` из библиотеки PubSubClient. Получаем:

```arduino
void loop() {
  if (!client.connected()) {
    reconnect();
  }
  client.loop();
}
```

Добавляем кусочек для публикации системных сообщений и получаем функцию loop целиком:

```arduino
void loop() {
  if (!client.connected()) {
    reconnect();
  }
  client.loop();

  long now = millis();  // в переменной now будет храниться время с момента запуска программы
  if (now - lastMsg > 30000) {    // если между временем последнего сообщения и текущим временем прошло более 30 секунд
    lastMsg = now;                // перезаписываем время в таймере последнего сообщения на текущее
    ++value;                      // счетчик +1
    snprintf (msg, sizeof(msg), "status #%ld", value);
    Serial.print("Publish message: ");
    Serial.println(msg);
    client.publish(mqtt_topic_status, msg);
  }
}
```
