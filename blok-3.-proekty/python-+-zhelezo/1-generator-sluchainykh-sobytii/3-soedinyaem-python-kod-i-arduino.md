## Получение данных от ардуино

Связывать Ардуино с компьютером мы привыкли по серийному порту Serial, так же и продолжим. В python есть библиотека, которая специально предназначена для работы с серийным портом.

Сначала библиотеку serial нужно установить. Для этого в терминале выполните:

```
pip install pyserial
```

Создайте новый файл, назовите его serial\_test.py. В начале мы должны сделать инициализацию соединения:

```python
import serial
import time

arduino = serial.Serial('COM3', 9600, timeout=1) # указываем свой номер порта и скорость
time.sleep(2)  # Ждём инициализации (как delay в ардуино)
```

Чтобы прочитать сообщение из серийного порта, можно использовать несколько методов (команд):

| Команда      | Что возвращает | Когда использовать                                                                                                    |
| ------------ | -------------- | --------------------------------------------------------------------------------------------------------------------- |
| `read(5)`    | bytes (байты)  | Когда знаем точный размер пакета. Команда из примера считает ровно 5 байт (число можно поменять, например `bytes(1)`) |
| `readline()` | bytes          | Для текстовых команд. Считывает строчку до символа переноса строки (`\n`)                                             |
| `read_all()` | bytes          | Чтобы считать вообще всю пришедшую информацию                                                                         |

&#x20;Мы будем использовать только первые два способа. `readlines()` - самый простой.  Но нужно запомнить, что если переноса строки не будет, то программа будет бесконечно ждать и не получать данные. Полученные данные из ардуино в питон файле с помощью такого способа выглядят так: `b'OK\n'`. Буква b в начале значит, что тип данных - byte (для питона нет разницы, а вот ардуино именно так отправляет).

Но принять строку будет недостаточно. Чтобы полноценно "вытащить" данные, нужно сделать еще несколько действий:

```python
# 1. Читаем строку (ждем символа конца строки \n)
raw = arduino.readline()       # например получили b'25.5\r\n'

# 2. Декодируем в строку
text = raw.decode('utf-8')     # в результате получим '25.5\r\n'

# 3. Убираем мусор (\r, \n, пробелы)
clean = text.strip()           # в результате получим '25.5'

# 4. Преобразуем в число (если нужно)
data = float(clean)           # в результате получим 25.5
```

То есть `прочитать -> декодировать -> убрать лишнее -> готово`&#x20;

Вместо 3-4 строк это можно записать в одну:

```cpp
data = arduino.readline().decode('utf-8').strip()
```

В результате чтение данных из серийного порта (то, что ардуино отправило в сторону питон программы) выглядит во так:

```python
while True: # бесконечный цикл
    if arduino.in_waiting > 0: # проверяем, есть ли новые данные
        data = arduino.readline().decode('utf-8').strip() # расшифровать пришедшее сообщение
        print(f"Получено: {data}")
```

Это бесконечный цикл, чтобы закончить выполнение программы, вам нужно кликнуть на терминал VS code, в котором выполняется программа, и нажать ctrl+c. Программа выдаст сообщение об ошибке и закончится.&#x20;

ЧТобы сделать выход из программы более красивым, можно добавить такую конструкцию:

```python
try: # Начинаем блок обработки исключений (попыток), чтобы корректно завершить работу при ошибке
    while True:
        if arduino.in_waiting > 0:
            data = arduino.readline().decode('utf-8').strip()
            print(f"Получено: {data}")
except KeyboardInterrupt:  # Перехватываем исключение, если пользователь нажал Ctrl+C для остановки
    print("\n Отключение...")
    arduino.close() # Корректно закрываем соединение с последовательным портом
```

## Отправка данных в ардуино

Выше вы научились получать значения от ардуино. В то же время, конечно, мы можем захотеть управлять какими-то устройствами и нам пригодится отправка данных из python кода на ардуино.

Ранее мы создали объект arduino вот таким образом:

```python
import serial
import time

arduino = serial.Serial('COM3', 9600, timeout=1) # указываем свой номер порта и скорость
time.sleep(2)  # Ждём инициализации (как delay в ардуино)
```

Теперь мы можем отправить значение в серийны порт ардуино вот так:

```python
arduino.write(b'1')  # Отправить байт '1'
```

Отправлять мы можем только байты. Варианты записи есть такие:

<table><thead><tr><th width="177">Что хотим отправить</th><th>Как написать в Python</th><th>Что улетит в порт</th></tr></thead><tbody><tr><td>Символ <code>'1'</code></td><td><code>arduino.write(b'1')</code></td><td>байт <code>0x31</code> (код символа '1')</td></tr><tr><td>Символ <code>'1'</code></td><td><code>arduino.write('1'.encode())</code></td><td>байт <code>0x31</code></td></tr><tr><td>Число <code>1</code></td><td><code>arduino.write(str(1).encode())</code></td><td>байт <code>0x31</code></td></tr><tr><td>Число <code>1</code> (неправильно)</td><td><code>arduino.write(1)</code></td><td>Ошибка: нужен <code>bytes</code>, не <code>int</code></td></tr><tr><td>Текст <code>"ON"</code></td><td><code>arduino.write(b'ON')</code></td><td>два байта: <code>0x4F 0x4E</code></td></tr></tbody></table>

Чтобы не запутаться, лучше использовать `arduino.write('1'.encode())`

(вместо '1' может быть и другое сообщение).

Со стороны Ардуино мы должна написать программу, которая будет слушать серийный порт и что-то делать в зависимости от пришедших команд. Пусть мы отправляем 1 или 0 и эти значения вкл\выкл лампочку (код инициализации и pinmode лампочки напишите сами):

```cpp
void loop() {
  if (Serial.available() > 0) { // если порт доступен и что-то в него пришло
    char cmd = Serial.read();   // сохранить полученное значение в переменной cmd (тип данных char)
    if (cmd == '1') {
      digitalWrite(lampa, 1);
    } else if (cmd == '0') {
      digitalWrite(lampa, 0);
    }
  }
}
```
