# Получение и передача нескольких значений между Arduino и Node-red

Ранее мы создавали системы, в которых отправлялось или принималось только **одно** значение. Например, управление **одной** лампой или **одним** мотором, или считывание значений только с **одного** датчика. Конечно, такие простые системы в реальности встречаются редко. В этой статье разберем, как отправлять и получать сразу **несколько значений за раз и расшифровывать их**.

Сложность заключается в том, что, если мы отправляем/получаем несколько значений за раз, мы должны иметь возможность **определить, к чему какое значение относится**. Например, если от ардиуно мы отправляем значение с датчика DHT - температуру и влажность - мы **не можем просто отправить два числа**, потому что **неизвестно**, что из них является темп., а что - влажностью. Так же и при управлении из Node-red, если, например, мы хотим управляет цветом RGB светодиода, нам нужно однозначно как-то указать, какое из чисел - это R, какое G и какое B.

В setup также нужно добавить Serial.begin(115200);

Теперь разбираемся с расшифровкой сообщений. В loop добавляем цикл `если Serail доступен`:

```cpp
void loop() {
  if (Serial.available()){
    
  }
}
```

Далее делаем следующее: командой `Serial.readStringUntil(символ)` можно прочитать пришедшую в Serial строку **до определенного символа**. Причем если несколько раз подряд выполнить эту команду, она выполнится последовательно - то есть сначала до одного символа, а потом оставшуюся часть до другого символа.&#x20;

Так и сделаем: у нас передаются команды `RGB_R:число` (или RGB\_G или RGB\_B). Будем читать сначала "команду" до двоеточия, а потом все оставшееся ("число") до символа переноса строки (потому что команды из Node red через Serial out отправляют символ переноса строки автоматически). Полученные "кусочки" сохраним в переменные key и val типа String:

```cpp
void loop() {
  if (Serial.available()){
    String key = Serial.readStringUntil(':'); //читаем часть сообщения до :
    String val = Serial.readStringUntil('\n'); //читаем часть сообщения после :
}
```

Далее остается написать только конструкцию if  с действиями, в зависимости от пришедшего ключа. Обратите внимание: val значение имеет тип String, чтобы передать его на светодиод, надо преобразовать в число командой `значение.toInt()`:

```cpp
void loop() {
  if (Serial.available()){
    String key = Serial.readStringUntil(':'); //читаем часть сообщения до :
    String val = Serial.readStringUntil('\n'); //читаем часть сообщения после :
    
    if (key == "RGB_R"){ //если команда RGB_R
      r = val.toInt(); // то число, пришедшее с этой командой, записать в переменную r
    } else if (key == "RGB_G"){
      g = val.toInt();
    } else if (key == "RGB_B"){
      b = val.toInt();
    }
}
```

Чтобы не загромождать код, можно сделать это в отдельной функции и назвать ее например processCommand().

В конце loop остается только зажечь светодиод цветами со значениями переменных r, g и b:

```c
void loop() {

  if (Serial.available()){
    String key = Serial.readStringUntil(':');
    String val = Serial.readStringUntil('\n');

    if (key == "RGB_R"){
      r = val.toInt();
    } else if (key == "RGB_G"){
      g = val.toInt();
    } else if (key == "RGB_B"){
      b = val.toInt();
    }
  rgb.setRGB(r, g, b);  // включаем светодиод
  }

}
```

## Управление из Node-red. Способ 2. JSON

В предыдущем способе мы отправляли из Node-red значения как строки в определенном формате `команда:значение`. А ведь для такого формата существует специальный формат обмена данными - json. Напомню:&#x20;

Для того, чтобы удобно формировать и принимать в ардуино сообщения в формате JSON, будем использовать библиотеку ArduinoJson (автор Benoit Blanchon).&#x20;

Пусть нам из Node-red приходит сообщение вида:

```json
{
    "red":"255",
    "green":"23",
    "blue":"100"
}
```

Если мы прочитаем его через Serial как строку (например в переменную s), то получим примерно это:

```c
s = "{\"red\":\"255\",\"green\":\"255\",\"blue\":\"255\"}"
```

Чтобы с помощью этой библиотеки "расшифровать" пришедшее значение в JSON формате:

```c
String r = s["red"]; // будет иметь значение "255"
String g = s["green"]; // будет иметь значение "23"
String b = s["blue"]; // будет иметь значение "100"
```

## Передача данных в Node-red. Способ 1

Аналогично сигналам управления, в нашей системе может находиться множество датчиков, присылающих сигналы для мониторинга. Задача со стороны Ардуино - отправить их в таком формате, чтобы затем мы могли в Node-red расшифровать сообщение и понять, где сигнал от какого датчика.

Попробуем настроить это на примере датчика DHT22. Он отправляет сразу два значения: температуру и влажность. Для начала прописываем в Arduino все, что нужно для подключения датчика:

```cpp
#include <DHT.h> // подклбчаем библиотеку DHT Sensor library
#include <DHT_U.h>

DHT dht(3, DHT22); // указываем пин подключения и тип датчика

void setup() {
  Serial.begin(115200); // не забываем подключить передачу данных по serial
  dht.begin(); // запускаем передачу с датчика
}

void loop() {

}
```

Внутри loop, очевидно, нужно считать значения температуры и влажности:

```cpp
void loop() {

  float temp = dht.readTemperature();
  float hum = dht.readHumidity();

}
```

Далее, все что нам нужно сделать - вывести значения в serial командой Serial.print, но с подписями и все в одну строку, чтобы сообщение выглядело так:

<mark style="color:green;">T:</mark>XX.XX,<mark style="color:green;">H:</mark>XX.XX<mark style="color:red;">\n</mark>

Здесь T и H - это ключи, по которым мы поймем, что за именно цифры стоять после. Двоеточия - это "отсечки", по ним поймем, что ключ закончился, со следующего символа уже начинается значение. \n в конце нужно добавить, чтобы обозначить конец сообщения. Это можно сделать командой println ли прямо написать \n  в print(). Сообщение составим из кусочков:

```cpp
void loop() {

  float temp = dht.readTemperature();
  float hum = dht.readHumidity();

  Serial.print("T:");
  Serial.print(temp);
  Serial.print(",H:");
  Serial.print(hum);
  Serial.println();
}
```

Единственное, что стоит добавить - это обернуть все в if, который будет проверять, не пустые ли значения от датчика (при неправильном подключении он может выдавать NaN вместо обоих значений):

```cpp
void loop() {

  float temp = dht.readTemperature();
  float hum = dht.readHumidity();

  if (!isnan(temp) && !isnan(hum)) { // проверка что значения не = NaN
    // Формат: T:23.5,H:45.0\n
    Serial.print("T:");
    Serial.print(temp);
    Serial.print(",H:");
    Serial.print(hum);
    Serial.println();
  }
}
```

То есть, если index == 0, то это температура, если index == 1, то это влажность. Добавим блок функции function после блока split:

<figure><img src="../../.gitbook/assets/image (187).png" alt=""><figcaption></figcaption></figure>

Эта функция будет находить сообщение с температурой по его index и вытаскивать само значение температуры из `"T:27.50"`. Напишем такой код:

```javascript
if (msg.parts.index ==0) { // если index внутри msg внутри parts == 0
    msg.payload = msg.payload.split(":")[1] // разделяем сообщение по двоеточию :
    return msg // пересылаем сообщение дальше
}
```

Самой непонятной частью является строчка:

```
msg.payload = msg.payload.split(":")[1]
```

msg.payload = ... -> это значит пересылаемое далее сообщение

msg.payload.split(":") -> команда split разделяет msg.payload (то есть пришедшее от предыдущего блока значения) по указанному в скобках значению, то есть ":". Обратно split выдает массив из двух получившихся кусочков, то есть \["T" , "27.50"]. Нас интересует второй кусочек, его индекс \[1]

Команда return msg -> это "отправить сообщение дальше". Мы вставляем return msg внутрь if, чтобы дальше сообщение шло, только если этот if выполнится.

Скопируйте получившуюся функцию и измените в копии занчение index в условии if на 1 для влажности:

```javascript
if (msg.parts.index == 1) {
    msg.payload = msg.payload.split(":")[1]
    return msg
}
```
