# Вывод в терминал на компьютере (монитор порта)

### Описание

Самый простой способ получить обратную связь от программы - вывод данных в **консоль** - окно ввода-вывода текстовых данных для взаимодействия с программой. Это самый простой способ отладки кода - отправлять из программы данные на разных участках её выполнения, выводить ошибки, показания датчиков и прочее.

Ранее мы попробовали выводить данные на дисплей, но часто такой вариант не подходит. Например, нет дисплея или возмжности его подключить. Или нужно выводить информацию просто для проверки чего-то (для отладки).

{% hint style="info" %}
&#x20;Например, вы подключили кнопку, написали код для управления светодиода от кнопки, но почему-то при нажатии кнопки код не срабатывает. Вопрос, который возникает - а работает ли кнопка? Или кнопка работает, а не работает светодиод? Может быть посмотреть, а приходит ли от кнопки вообще сигнал? Но не подключать же для этого экран... Вывести бы куда-нибудь значение от кнопки и проверить...
{% endhint %}

Для таких целей мы можем использовать монитор порта в Arduino IDE

<details>

<summary>Почему монитор и почему порта? Почему не терминал\консоль?</summary>

С Arduino ситуация немного не такая, как с программой для компьютера - программа выполняется на отдельном устройстве со своим процессором, к которому нет прямого доступа. Arduino подключается к компьютеру по USB, на самой плате USB гнездо подключено не напрямую к микроконтроллеру - он не умеет с ним работать - а к USB-UART преобразователю. **UART** - это универсальный приёмник-передатчик, почти все микроконтроллеры имеют такой интерфейс для связи с внешними устройствами. UART представлен двумя пинами **RX** _(Receive, приём)_ и **TX** _(Transmit, передача)_ - именно к ним и подключен преобразователь. Взглянем на плату Arduino Nano:

[![](https://alexgyver.ru/wp-content/uploads/2024/12/ardu-uart-600x378.png)](https://alexgyver.ru/wp-content/uploads/2024/12/ardu-uart.png)

С его помощью Arduino может принимать и отправлять данные по USB проводу.

#### COM порт <a href="#id-2-toc-title" id="id-2-toc-title"></a>

Компьютер взаимодействует именно с USB-UART преобразователем, он не знает, что это плата Arduino. При подключении Arduino на стороне компьютера создаётся виртуальный **COM порт** (последовательный порт) - именно так компьютер видит нашу плату:

![](https://alexgyver.ru/wp-content/uploads/2024/12/usb-com.png)

То есть на стороне микроконтроллера для связи используется UART, а на стороне компьютера - COM порт.

#### Монитор порта <a href="#id-3-toc-title" id="id-3-toc-title"></a>

Чтобы работать с портом на стороне компьютера, понадобится **монитор порта** - программа, которая умеет отправлять и читать данные с COM порта. Таких программ много и подойдёт любая, но в Arduino IDE есть свой встроенный монитор порта, что очень удобно.&#x20;

В окне монитора порта есть несколько важных настроек:

* **Скорость** - скорость связи. Должна совпадать со скоростью UART, установленной в программе на микроконтроллере. Обычно `9600` или `115200` бод
* **Конечный символ** или терминирующий символ, **конец строки** - будет добавляться в конце текста при отправке из окна монитора порта

### Отправка в монитор порта

Для работы с портом на стороне Arduino используется системный объект `Serial`, он позволяет читать и отправлять данные.&#x20;

Перед началом работы нужно запустить связь с указанием скорости - команда `Serial.begin(скорость)` внутри `setup()`:

```arduino
void setup() {
    Serial.begin(115200);   // запустить связь на скорости 115200 бод
}
```

`Serial` может отправлять данные из программы в монитор порта или на другие МК и микросхемы, для этого используются команды `print(любые данные)` и `println(любые данные)` - первая просто печатает данные, вторая печатает и переносит строку. Давайте напишем классический пример, который выводит строку `"Hello, World!"` в консоль:

```cpp
void setup() {
    Serial.begin(115200);
    Serial.println("Hello, World!");
}

void loop() {
}
```

Загрузите программу и откройте монитор порта на скорости `115200`:

![](https://alexgyver.ru/wp-content/uploads/2024/12/hello.png)

Данный код выполнится один раз при запуске МК (потому что команда написана в функции `setup`). Нажмите на плате кнопку Reset (перезагрузка) несколько раз - после каждого нажатия в порт снова выведется строка.

Если перенести `Serial.println("Hello, World!")` в `loop`, то в монитор порта постоянно будет прилетать это сообщение.

### Задание для самостоятельного выполнения (вывод)

Подключите кнопку к ардуино. Напишите код, который выводит в серийный порт состояние кнопки.

### Чтение из порта

Также мы можем в нашу программу считывать данные из монитора порта (ведь по серийному порту могут прийти, например, данные от другого микроконтроллера, или пользователь может сам там ввести какие-то команды).

Для проверки наличия входящих данных используется метод `available()` - он вернёт количество входящих байт. В таком условии можно проверить, что есть хотя бы один байт для чтения:

```arduino
void loop() {
    if (Serial.available()) {
        // есть входящие данные
    }
}
```
