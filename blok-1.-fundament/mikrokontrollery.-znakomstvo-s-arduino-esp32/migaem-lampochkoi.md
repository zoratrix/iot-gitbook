# Мигаем лампочкой

## Сборка

Для сборки первой схемы нам понадобятся:

1\) **Светодиод** (по английски – LED) – маленькая лампочка. Загорится, если подать ток. У светодиода две ножки, большая и маленькая. **Большая** = к <mark style="color:$danger;">**(+)**</mark> питания, **маленькая** = к <mark style="color:blue;">**(-)**</mark> питания. Легко запомнить по первой букве: **м**аленькая = **м**инус. Если перепутать ножки, то светодиод сгорит.

<figure><img src="../../.gitbook/assets/image (126).png" alt=""><figcaption></figcaption></figure>

**Резистор** – ограничивает ток. Цветные полоски указывают на величину сопротивления.&#x20;

<figure><img src="../../.gitbook/assets/image (127).png" alt=""><figcaption></figcaption></figure>

Чем **больше сопротивление**, тем **меньше тока** пройдет через резистор. Но, если слишком сильно уменьшить ток, то его будет "недостаточно" и лампочка будет гореть слишком тускло.

<details>

<summary>Расчет резистора</summary>

Если отвественно подойти к вопросу, то можно рассчитать, какой именно резистор нужен для нашего светодиода. Для этого нужно воспользоваться законом Ома:

$$R = U/I$$

где R = сопротивление, U = напряжение, I = ток.

Чтобы вычислить R, нам нужно узнать U и I. I (ток) можно посмотреть в интернете, светодиоды бывают разными, самые распространенные - на 20 мА (0,02 А). Напряжение же посчитаем как разницу между напряжением питания _Uпит_ (от ардуины 5 В в нашем случае) и напряжения самого светодиода _Uсвет_ (так же смотрим в интернете, обычно от 2 до 4 В, для примера возьмем 3 В).

Итого:

$$R =\frac{U_{пит} - U_{свет}} {I} = \frac{5 - 3} {0.02} = 100$$ Ом

## Программирование

В простейшем случае код программы для контроллеров Arduino делится на три основные части.

<figure><img src="../../.gitbook/assets/image (87).png" alt=""><figcaption></figcaption></figure>

Часть A, которая находится в самом начале кода:

* Выполняется 1 раз при запуске
* Назначить имена и адреса пинов
* Подключить библиотеки
* Инициализировать переменные

Часть B, функция `void setup()`:

* Выполняется 1 раз при запуске
* Назначить режимы работы пинов
* Прописать настройки (например логин и пароль от WiFi)

Часть C, функция `void loop()` - основной  код программы:

* Выполняется постоянно
* Команды, которые будут выполняться, пока контроллер включен

Переходим к программе для мигания.&#x20;

Самая первая вещь, которую нужно сделать – это сообщить программе, что и куда подключено. В **3-ий** **пин** подключен светодиод. Назовем его <kbd>lampa</kbd>. Назначение адреса выполняется командой `#define` перед циклом setup (то есть на самой первой строчке):

```arduino
#define lampa 3

void setup() {
  
}

void loop() {
                   
}
```

Далее  в цикле функции `setup()` нужно объявить, в каком режиме используется этот пин - как **входной** или **выходной**.&#x20;

{% hint style="info" %}
**Вход** контроллера позволяет ему **получать** информацию от какого-то элемента. Ко входам подключаются все датчики всех типов и выключатели. Выключатель или датчик даёт сигнал о нажатии или о срабатывании, контроллер его получает, значит, для контроллера это вход.

Контроллер не посылает на датчики и выключатели никаких сигналов, только получает сигналы от них.
{% endhint %}

{% hint style="info" %}
**Выход** контроллера позволяет контроллеру **выдавать** сигнал. То есть, управлять чем-то. Например, светом, шторами, приводами радиаторов, сиреной, дверным замком, розетками - чем угодно, подавая на него напряжение или замыкая контакты
{% endhint %}

Для этого используется функция `pinMode(название или номер пина, режим работы)`. Режим работы - INPUT (входной) или OUTPUT (выходной). Получается команда `pinMode(LED_BUILTIN, OUTPUT);`

```arduino
#define lampa 3

void setup() {
  pinMode(lampa, OUTPUT); // лампа - выходной сигнал
}

void loop() {
              
}
```

Далее в цикле `loop()` нужно написать, что, собственно, должен делать этот светодиод.&#x20;

**Мигание** - это попеременное включение\выключение с небольшой паузой между ними.&#x20;

* Чтобы включить светодиод, нужно отправить на пин, к которому он подключен, "высокий" электрический сигнал (1 или HIGH).&#x20;
* Чтобы выключить, нужно отправить "низкий" сигнал (0 или LOW).&#x20;

Чтобы отправить на пин какой-нибудь сигнал, используется функция

&#x20;`digitalWrite(название пина, какой сигнал отправить)`.&#x20;

Чтобы **выключить** встроенный светодиод, нужно написать:

<mark style="color:blue;">`digitalWrite`</mark>`(lampa, 0);` или <mark style="color:blue;">`digitalWrite`</mark>`(lampa, LOW);`

Соответственно, чтобы **включить**:

<mark style="color:blue;">`digitalWrite`</mark>`(lampa, 1);` или <mark style="color:blue;">`digitalWrite`</mark>`(lampa, HIGH);`

Код, чтобы просто **включить** светодиод:

```arduino
#define lampa 3

void setup() {
  pinMode(lampa, OUTPUT); // лампа - выходной сигнал
}

void loop() {
  digitalWrite(lampa, 1); //включили светодиод        
}
```

Мы уже включили светодиод, значит нужно **подождать**, а потом его выключить. А потом снова подождать и включить. И так далее. если не сделать паузу, а написать вкл-выкл подряд, то мы не увидим мигание - оно будет слишком быстрым.

«Подождать», то есть сделать задержку, можно командой `delay(время)`. В скобках пишется время в миллисекундах (то есть секунды, умноженные на 1000)

Итак, после включения и после выключения нужно поставить паузы, например в 1 секунду.

В результате весь код будет выглядеть следующим образом:

```arduino
#define lampa 3

void setup() {
  pinMode(lampa, OUTPUT); // лампа - выходной сигнал
}

void loop() {
  digitalWrite(lampa, 1);  // включаем светодиод
  delay(1000);             // ждем 1 секунду
  digitalWrite(lampa, 0);  // выключаем светодиод
  delay(1000);             // ждем 1 секунду
}
```

Остается загрузить программу в контроллер.
